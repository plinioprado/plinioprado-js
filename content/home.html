Business and Finance Projects

Current Financial and Business control prototypes - CTR series

Series of public Proofs Of Concept experimenting with code approaches and technologies that can be used for actual professional projects.

Used as a base for future prototypes.

  • The front-end is in ctr-client-react-base a rewrite from zero of ctr-client-react using the latest React standards and containing componenta able to handle multiple auxiliary tables based in data+format REST responses.
  • The back-end is in ctr-server-node-base a rewrite from zero of ctr-server-node using an MVC approach able to provide data+format REST responses.

Click here for more detailed requirements and development notes.

The current status is:

  • The master branch (main in the server repo) contains a prototype that is evolving but already working for the base.
  • The devdoc branch (in both repos) contains a work in progress for using the Base for an application that will handle Invoices with multiple Counterparts (People), Countries, Currencies.... Node was the choice to model it faster this same modeling can be coded in any OOP language.

Previous CTR series prototypes

The other POCs are REST API back-ends, where the Business Logic Layers use different languages and Data Access Layers use different SQL and NoSQL databases:

  • ctr-server-java in Java/Spring + a hardcoded db, it's the base for a private fork with PostgreSQL.
  • ctr-server-php in plain PHP + MySQL.
  • ctr-server-node in Node.js/Express + Firebase/MongDB/PostgreSQL/json.

Some are front-end projects spread from older Angular 1 and 2 to more recent ones using React.

  • ctr-client-react controls a basic invoice in React.
  • ctr-prefixedloans calculates a Brazilian pre-fixed loan in Angular 2.
  • ctr-client-angular controls Invoices and generation of account receivables in Angular 1.

The complete series can be found in public repos whose names start with "ctr" at www.https://github.com/plinioprado.
